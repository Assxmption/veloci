> VELOCI watches 8 platforms simultaneously, spots what's about to blow up **12вҖ“72 hours before it peaks**, and auto-generates ranked video scripts for it вҖ” so you're always the creator who posted first, not the one playing catch-up.

## Why this exists

Every content creator faces the same problem: by the time you *notice* something is trending, it already is. You're not early вҖ” you're noise.

The insight that drove VELOCI:

- Reddit **RISING** posts (not HOT) show velocity, not popularity
- Google Trends **RELATED RISING** queries hit breakout (5000%+ growth) 24вҖ“72h before a topic dominates YouTube search
- GDELT covers 100+ languages вҖ” a launch in Korea surfaces in English feeds hours later

**Lead time is the only moat in content creation.** VELOCI is that moat.

## Scoring explained

Each trend gets a **composite score** from 0 вҶ’ 1, then bucketed into a tier:

| Tier | Score | What it means | What you should do |
|------|-------|---------------|-------------------|
| рҹҹў **early** | 0.70вҖ“1.00 | You're 12вҖ“48h ahead of mainstream | Post in the next 2вҖ“6 hours |
| рҹҹЎ **emerging** | 0.50вҖ“0.70 | Window is open, closing fast | Post within 24 hours |
| рҹҹ  **trending** | 0.35вҖ“0.50 | Viable but saturating | Proceed with caution |
| рҹ”ҙ **saturated** | < 0.35 | Everyone's already covered it | Hard skip |

The velocity score is the most critical signal вҖ” we care about **rate of acceleration**, not current popularity. A rising post with 200 upvotes and 40/min velocity beats a hot post with 2000 upvotes and 3/min velocity every time.

## Setup

```bash
git clone https://github.com/yourusername/veloci.git
cd veloci

pip install -r requirements.txt
python -m spacy download en_core_web_sm
python -m nltk.downloader stopwords punkt
```

### 2. Configure API keys

All keys have generous free tiers вҖ” this whole system runs at **$0/month**.

```bash
cp .env.example .env
```

Then fill `.env`:

```env
REDDIT_CLIENT_ID=your_id          # reddit.com/prefs/apps вҶ’ script app
REDDIT_CLIENT_SECRET=your_secret
REDDIT_USER_AGENT=veloci-bot/1.0 by u/yourusername

YOUTUBE_API_KEY=your_key          # console.cloud.google.com вҶ’ YouTube Data API v3

TWITTER_BEARER_TOKEN=your_token   # developer.twitter.com вҶ’ bearer token

NEWS_API_KEY=your_key             # newsapi.org вҶ’ free (100 req/day)

APIFY_TOKEN=your_token            # console.apify.com вҶ’ for Instagram (optional)
```

> Google Trends, GDELT, and TikTok need **no keys** вҖ” they just work.

## Why these design choices?

**DBSCAN over K-means** вҖ” K-means needs K upfront. We don't know how many clusters will emerge from a given cycle. DBSCAN discovers them organically and marks true outliers as noise rather than forcing them into the wrong group.

**RISING not HOT** вҖ” Reddit's rising sort shows *velocity right now*. Hot shows *what already won*. The delta between them is your lead time.

**GDELT** вҖ” Scrapes 100+ languages, 250+ countries, near-real-time. A product launch in Korea or a regulation in Brazil appears in GDELT hours before it trends in English social media. For international niches this is the unfair edge nobody talks about.

**Google Trends RELATED RISING** вҖ” Breakout queries (5000%+ growth) are the single best predictor of what people will be searching for on YouTube in 24вҖ“72 hours. This is the upstream signal that most trend tools don't even know to look for.

**No LLM in Stage 02** вҖ” The script generator uses an intelligent template engine seeded with actual trend data (entities, keywords, sentiment, platform context). It's fast, deterministic, free, and produces scripts that are already 80% of the way there. Plug in an LLM call to `_generate_body()` if you want the other 20%.

**Instagram by design, not by accident** вҖ” IG has no public trend API and its private API violates ToS. But IG trends lag 2вҖ“5 days behind TikTok/Reddit anyway. VELOCI predicts IG trends *before they exist on IG* вҖ” which is the entire point.

## Known limitations (being honest)

1. **TikTok will occasionally break** вҖ” they actively change their DOM. The Playwright scraper uses CSS selectors + XHR intercept as dual fallbacks. The XHR path is more stable. Expect to update selectors every few months.

2. **Twitter free tier is real** вҖ” 500K tweets/month sounds like a lot. At 100 results Г— 5 queries Г— 48 cycles/day you're at 240K/day. Scale down queries or move to Basic ($100/mo) if you hit the wall.

3. **Google Trends 429s** вҖ” pytrends is unofficial. The 2.5s inter-request delay is conservative and should be fine. If you see 429s, bump `INTER_REQUEST_DELAY` in `config.py`.

4. **Embedding on cold start is slow** вҖ” `all-MiniLM-L6-v2` downloads ~85MB on first run and loads into RAM (~200MB). Subsequent runs are fast. On a potato machine, give it 30вҖ“60s on first cycle.
